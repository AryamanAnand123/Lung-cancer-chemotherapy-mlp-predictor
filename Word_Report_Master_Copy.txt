CHAPTER 1
INTRODUCTION
========================================

1.1 Background
Lung cancer remains one of the leading causes of cancer mortality worldwide. In many clinical settings, chemotherapy is still a primary treatment option, especially for patients without accessible targetable molecular markers. In this context, early prediction of mortality risk and treatment response can improve care planning, patient monitoring, and decision support.

1.2 Project Overview
This project presents a dual-task machine learning framework for:
(1) one-year mortality prediction, and
(2) binary chemotherapy treatment-response prediction.
The system compares LightGBM, XGBoost, and TabNet under nested cross-validation and produces calibrated, explainable, and deployment-ready outputs.

1.3 Motivation
Many published models rely on proprietary institutional biomarkers and are hard to reproduce in practical settings. This work is motivated by the need for a reproducible and auditable approach using public-data-compatible variables while maintaining strong predictive quality.

1.4 Problem Statement
Current clinical ML pipelines often show:
- weak reproducibility,
- overreliance on AUC only,
- limited calibration and threshold analysis,
- poor train-to-deployment continuity.
This project addresses these issues using traceable report artifacts, uncertainty-aware metrics, and deployment auto-selection logic.

1.5 Objectives
- Build a unified dual-task prediction framework.
- Train and compare LightGBM, XGBoost, and TabNet.
- Evaluate with AUC, F1, Brier score, CI, calibration, and threshold optimization.
- Generate explainability artifacts (SHAP for tree models).
- Implement report-driven model deployment selection.

========================================
CHAPTER 2
LITERATURE REVIEW
========================================

2.1 Machine Learning for Lung Cancer Mortality and Survival Prediction
Literature shows that ML can improve prognosis modeling in lung cancer, but many studies rely on private clinical variables and limited cohort generalizability. Some studies report strong AUC/C-index but include methodological caveats such as aggressive model search and possible performance optimism.

2.2 Machine Learning for Chemotherapy Response and Complication Prediction
Response and post-chemotherapy complication studies demonstrate potential for ML-based triage support. However, endpoint definitions vary significantly across papers, making direct comparison difficult. Small labeled response cohorts further increase model instability risk.

2.3 Explainability, Calibration, and Deployment in Clinical ML
Recent clinical AI practice emphasizes that discrimination alone is insufficient. Reliable systems should include calibration checks, uncertainty reporting, threshold analysis, and explainability. This project aligns with that direction by integrating SHAP, reliability diagnostics, and deployment logic tied to validated report outputs.
